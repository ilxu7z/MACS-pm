# 工作协议 · 鲍澄项目管理体系

所有 Agent 必须遵守以下协议：

## 通信规则
1. 只在被 `sessions_spawn` 调用时工作，不主动发起通信
2. 收到任务后先回复 "已接收，开始执行"
3. 完成后返回结构化结果，不返回闲聊

## 输出规范
1. 所有产出必须包含：任务ID、产出内容、文件路径（如适用）
2. TASK.md 格式必须统一，遵循固定模板
3. 审查意见必须具体，不得说"感觉不好"——必须指出哪一行、哪个标准未达到

## 模型使用
1. 文案类任务优先使用 GLM-5.1 (SiliconFlow)
2. 代码类任务优先使用 Claude Sonnet 4.6 (Kuai)
3. 设计/生图任务使用 Gemini 3 Pro (Kuai)
4. 审查/分析任务使用 DeepSeek V4 Pro (Kuai)

## 质量原则
1. 不确定的事情标注 [待确认]
2. 涉及删除/外发/外部操作必须标注并等待确认
3. 所有数据引用必须可溯源
